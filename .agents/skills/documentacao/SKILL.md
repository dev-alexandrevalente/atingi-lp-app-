# Co-Autoria de Documentação

## Quando Usar
- Escrevendo READMEs, guias de setup, documentação técnica
- Melhorando documentação existente
- Criando guias para contribuidores
- Documentando APIs ou arquitetura

## Processo

### Etapa 1: Coleta de Contexto
1. **Perguntas iniciais**: Qual o público? Qual o objetivo? Qual o formato?
2. **Dump de informações**: Reúna todo material bruto disponível

### Etapa 2: Refinamento e Estrutura

1. **Perguntas de clarificação**: Esclareça ambiguidades
2. **Brainstorming**: Gere tópicos e seções
3. **Curadoria**: Selecione e ordene o conteúdo
4. **Verificação de lacunas**: Identifique informações faltantes
5. **Rascunho**: Escreva a primeira versão
6. **Refinamento iterativo**: Melhore com feedback

### Etapa 3: Teste de Leitura

1. **Preveja perguntas do leitor**: O que ficaria confuso?
2. **Teste**: Leia como se fosse a primeira vez
3. **Checagens**: Linguagem clara? Exemplos suficientes? Estrutura lógica?
4. **Itere**: Corrija baseado nos achados

## Revisão Final
- [ ] Propósito claro no início
- [ ] Estrutura lógica e escaneável
- [ ] Exemplos práticos incluídos
- [ ] Sem jargão desnecessário
- [ ] Links e referências funcionam
- [ ] Formatação consistente

## Dicas
- Prefira frases curtas e diretas
- Use listas e tabelas para informação estruturada
- Inclua exemplos de código quando relevante
- Mantenha consistência de estilo
