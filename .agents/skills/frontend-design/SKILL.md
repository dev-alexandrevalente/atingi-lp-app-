# Design de Frontend (Premium e Produção)

## Mandato Central
Cada interface deve ser **visualmente distintiva e de nível profissional**. Interfaces genéricas são inaceitáveis.

## Quando Usar
- Criando novas interfaces de usuário
- Redesignando componentes existentes
- Implementando sistemas de design
- Melhorando qualidade visual e interações

## Fase de Design Thinking (Obrigatória)

### 1. Propósito
Antes de implementar, responda: Qual é o objetivo desta interface? Quem vai usar? Em que contexto?

### 2. Tom (Escolha uma direção dominante)
- Elegante e minimalista
- Vibrante e energético
- Profissional e confiável
- Moderno e sofisticado

### 3. Âncora de Diferenciação
O que torna esta interface **única**? Identifique 1-2 elementos que a distinguem.

## Regras de Execução Estética (Inegociáveis)

### Tipografia
- Use fontes modernas (Google Fonts: Inter, Roboto, Outfit)
- Hierarquia clara de tamanhos
- Espaçamento adequado

### Cor e Tema
- Paletas curadas e harmônicas (nunca vermelho/azul/verde genéricos)
- Dark mode com profundidade (não apenas fundo preto)
- Gradientes suaves para dimensão

### Composição Espacial
- Espaço em branco generoso
- Grid consistente
- Alinhamento impecável

### Movimento
- Micro-animações para feedback
- Transições suaves (200-300ms)
- Efeitos hover significativos

### Textura e Profundidade
- Sombras sutis para hierarquia
- Glassmorphism quando apropriado
- Bordas e separadores discretos

## Anti-Padrões (Falha Imediata)
- ❌ CSS inline excessivo sem sistema
- ❌ Cores do browser default
- ❌ Sem hierarquia visual
- ❌ Layouts não responsivos
- ❌ Animações excessivas que prejudicam performance

## Checklist do Operador
- [ ] Design Thinking completo
- [ ] Sistema de cores definido
- [ ] Tipografia consistente
- [ ] Responsivo em mobile/tablet/desktop
- [ ] Micro-animações implementadas
- [ ] Acessibilidade verificada
