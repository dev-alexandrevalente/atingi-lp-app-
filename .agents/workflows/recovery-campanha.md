# Workflow: Recovery de Campanha

## Objetivo
Criar sistema de recuperação multi-canal para leads que abandonaram o funil.

## Gatilho de Uso
- Lead abandona checkout
- Lead abandona quiz/fluxo
- No-show de call/booking
- Base de leads fria que precisa ser reativada
- Alto abandono em qualquer etapa

## Skills Envolvidas
1. `analytics-strategist`
2. `recovery-architect`
3. `copywriting-engine`

## Sequência de Handoff

### Fase 1 — Diagnóstico de Abandono
```
analytics-strategist → Identificar ONDE os leads abandonam e QUANTOS
  ↓ output: mapa de abandono com volume por etapa
```

### Fase 2 — Arquitetura de Recovery
```
recovery-architect → Projetar sistema de recuperação (5 fases × N canais)
  ↓ output: sequência completa com timing, canais, e copy
```

### Fase 3 — Copy
```
copywriting-engine → Refinar copy de cada mensagem (email, WhatsApp, SMS)
  ↓ output: copy final por fase e canal
```

### Fase 4 — Mensuração
```
tracking-implementation → Garantir tracking de recovery (aberturas, cliques, retornos)
analytics-strategist → Definir métricas de sucesso do recovery
```

## Output Esperado
- [ ] Mapa de abandono com priorização
- [ ] Sistema de recovery com 5 fases
- [ ] Copy de todas as mensagens por canal
- [ ] Cadência de timing definida
- [ ] Tracking configurado
- [ ] Benchmarks de sucesso definidos
