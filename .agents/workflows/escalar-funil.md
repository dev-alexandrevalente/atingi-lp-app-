# Workflow: Escalar Funil

## Objetivo
Escalar um funil que já funciona sem quebrá-lo — aumentar volume mantendo ou melhorando taxa de conversão e ROAS.

## Gatilho de Uso
- Funil validado com ROAS positivo
- Momento de aumentar budget
- Expansão para novos públicos/plataformas
- Otimização de LTV para permitir CPA maior

## Skills Envolvidas
1. `analytics-strategist`
2. `growth-engine`
3. `cro-optimizer`
4. `creative-director`
5. `post-purchase-architect`
6. `recovery-architect`

## Sequência de Handoff

### Fase 1 — Diagnóstico de Saúde
```
analytics-strategist → Análise de cohorts + curvas de retenção + unit economics
  ↓ output:
  - CPA atual
  - ROAS por canal
  - LTV por cohort
  - Margem para escala
  - Gargalos de escala identificados
```

### Fase 2 — Growth Plan
```
growth-engine → Estratégia de escala: canais, budget, timing
  ↓ output:
  - Plano de escala mensal (mês 1, 2, 3)
  - Budget allocation por canal
  - Testes de público/criativo priorizados
```

### Fase 3 — Creative Refresh
```
creative-director → Novos ângulos + hooks para escala
  ↓ output:
  - 5+ novos criativos
  - Plano de teste criativo
  - Estratégia anti-fadiga
```

### Fase 4 — Otimização de Revenue
```
cro-optimizer → Otimizar taxa de conversão no funil existente
post-purchase-architect → Melhorar onboarding e reduzir refund
recovery-architect → Implementar/otimizar sistema de recovery
  ↓ output combinado:
  - Teste A/B no funil
  - Pós-compra otimizado
  - Recovery configurado
```

## Output Esperado
- [ ] Unit economics documentados
- [ ] Plano de escala em 3 meses
- [ ] Novos criativos produzidos
- [ ] Funil otimizado com testes A/B
- [ ] Recovery implementado
- [ ] Pós-compra reforçado
- [ ] Learnings registrados em `memory/learnings/`
