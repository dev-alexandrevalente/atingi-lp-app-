# 🔁 Patterns — Memory System

## Papel
Padrões recorrentes validados: sequências, estruturas, templates e fórmulas que funcionam repetidamente. É o playbook vivo do ecossistema.

## Quem Escreve
- Qualquer skill que identifique um padrão repetido
- `/orquestrador` ao consolidar ciclos

## Quem Lê
- Skills que precisam de referência antes de criar algo novo
- `/orquestrador` para roteamento informado

## Formato de Arquivo
```
pattern_[dominio]_[nome-curto].md
```

## Estrutura Obrigatória
```markdown
# Pattern: [Nome do Padrão]
- **Domínio**: [copy | funil | criativo | analytics | etc.]
- **Validado em**: [quantas vezes / projetos]
- **Confiança**: [alta | média | baixa]

## Descrição
[o que é este padrão em 2-3 linhas]

## Quando Usar
[situações em que este padrão se aplica]

## Estrutura
[passo a passo ou template do padrão]

## Exemplos
[1-2 exemplos concretos de uso bem-sucedido]

## Contra-indicações
[quando NÃO usar este padrão]
```
