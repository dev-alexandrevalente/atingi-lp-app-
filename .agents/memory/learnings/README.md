# 📚 Learnings — Memory System

## Papel
Aprendizados consolidados de campanhas, testes A/B, diagnósticos e sprints. Cada learning é uma lição validada que evita repetir erros e acelera decisões futuras.

## Quem Escreve
- `analytics-strategist` (após análise de dados)
- `cro-optimizer` (após testes A/B)
- `creative-director` (após iterações criativas)
- Qualquer skill ao final de um ciclo via `/orquestrador`

## Quem Lê
- Todas as skills que precisam de contexto histórico
- `/orquestrador` (para decisões informadas)

## Formato de Arquivo
```
learning_[YYYY-MM-DD]_[tema-curto].md
```

## Estrutura Obrigatória
```markdown
# Learning: [Título Descritivo]
- **Data**: [YYYY-MM-DD]
- **Projeto**: [nome do produto/funil]
- **Skill origem**: [qual skill gerou]
- **Tipo**: [teste A/B | diagnóstico | campanha | sprint | outro]

## Contexto
[o que estava sendo feito]

## Hipótese
[o que se esperava]

## Resultado
[o que aconteceu — com dados]

## Lição
[o que aprendemos — frase clara]

## Ação Futura
[o que fazer diferente na próxima vez]

## Tags
[lista de tags: #copy #hero #checkout #criativo #etc]
```
